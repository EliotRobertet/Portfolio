Assignment 1

The making of this website

Overview of the First Assignment in VÉL608G – Modern Manufacturing Processes

The initial assignment for the course VÉL608G Modern Manufacturing Processes involved creating a website to serve as a portfolio for the course's assignments while gaining familiarity with Git. This website showcases the process of its creation, detailing the steps taken and challenges encountered along the way.

Preparation and Research

Before starting the website, I carefully reviewed the assignment description, watched related tutorial videos on YouTube, and studied a comprehensive guide on HTML available at w3schools. These resources provided a solid foundation for understanding the requirements and technical aspects of the project.

With a clear vision of how I wanted the website to appear, I proceeded to download the recommended tools for the assignment. I installed Brackets, a user-friendly code editor offering a live preview feature, which was instrumental in the website's development. Additionally, I created a GitHub account to host the website and installed Git Bash from Git to upload the project files to a repository.

Youtube video of the teacher's instructions

Development Process

Once the preparation phase was complete, I focused on selecting a suitable website template from HTML5UP. My primary goal was to find a template that was simple, visually appealing, and functional, including a drop-down menu to organize and access all assignments. I selected the Landed template, which aligned perfectly with these criteria. I downloaded the template and began refining it using Brackets. The customization process included:

  • Removed unnecessary sections from the homepage and added relevant ones to meet the project requirements.
  • Created individual HTML files for each subpage and linked them appropriately in the drop-down menu and "Learn More" buttons.
  • Updated all placeholder texts on the homepage and menu to reflect the purpose and content of the portfolio.
  • Linked footer icons to their corresponding destinations on every page.

Image Editing and Processing

To prepare images for the website, we primarily used the Snipping tool in Windows to capture screenshots. Afterward, we used the standard image editing tool in Windows to resize and adjust the images to fit the layout of the website. This ensured that the images loaded quickly while maintaining quality.

Additional Tools and Resources

While working on the assignment, we discovered and used Phoenix, which is the new version of Brackets. It provided a user-friendly interface and helpful features for coding and refining the website. The most helpful feature being live preview which allows you to edit the html in real time, in the same window. Also when double clicking in the html file you could locate that text/button or image in the html code, which made editing the code a breeze.

Phoenix is a modern, updated version of Brackets, providing a clean and intuitive interface for coding and refining projects like this website.

The CV

The assignment required linking a CV to the website. To achieve this, I uploaded my CV in PDF format to the project folder and created a button on the homepage that links to the file. Below is the HTML code used to implement the button:


<a href="cv.pdf" class="button primary" target="_blank">View My CV</a>
View My CV

When the button is clicked, it opens the CV in a new tab, ensuring easy access without navigating away from the main website. This approach provides a professional and user-friendly way to share the CV with visitors.

Publishing the website

After completing the design of my website, I published it by uploading the project to GitHub using Git Bash. First, I created a repository on GitHub, set it to public, and assigned it an appropriate name. Next, I opened Git Bash, which I had installed during the preparation phase, directly in the website’s directory. To do this, I right-clicked inside the website’s folder and selected "Open Git Bash here."

Here you can see that you have to right click in the folder and select: "Open Git Bash here"

Uploading the Website to GitHub Using Git Bash

Below is a detailed step-by-step explanation of how I uploaded my website to GitHub using Git Bash. Each step is accompanied by the corresponding command or action.

This is a picture of the Git Bash code, which will be described later on.

Step 1: Initialize a Git Repository

To begin, I opened Git Bash in the folder containing my website files and initialized a Git repository. This sets up Git tracking for the folder.


git init

Step 2: Link the Repository to GitHub

I created a new repository on GitHub and linked it to my local repository using the following command:


git remote add origin https://github.com/EliotRobertet/Portfolio.git

To confirm the link, I checked the remote repositories with:


git remote -v

Step 3: Stage All Files

I staged all files in my directory to prepare them for the initial commit. The command I used ensures all files in the directory are included:


git add .

Step 4: Commit the Files

I committed the staged files with a descriptive message. Initially, I encountered an error due to the absence of author identity configuration:


git commit -m "Initial commit: Upload website files"

To resolve the issue, I set my global username and email configuration:


git config --global user.name "EliotRobertet"
git config --global user.email "eliot210203@gmail.com"

After setting the identity, I successfully committed the files:


git commit -m "Initial commit: Upload website files"

Step 5: Push the Files to GitHub

Finally, I pushed the committed files to the GitHub repository using the following command:


git push -u origin master

Step 6: Verify the Upload

I visited my repository on GitHub (https://github.com/EliotRobertet/Portfolio) to confirm that all files were successfully uploaded.

Conclusion

This process efficiently uploaded my website files to GitHub while maintaining version control using Git. The commands and configurations ensure seamless updates in the future.

Deploying the page

After successfully pushing the files using Git Bash, they will appear in the repository you created on GitHub. To publish the website, navigate to Settings > Pages in the repository. Ensure that the main branch is selected and that the root folder is specified. Once these settings are saved, a link to your live website will be generated. If it does not appear immediately, refresh the settings page to view the link.

The master branch is selected and the root folder.

How We Updated the Website Using Git Bash

As part of the assignment, we learned how to update the website efficiently using Git Bash. Below is a step-by-step explanation of how we pushed updates to the repository and deployed them to the live site:

  1. Opened Git Bash: After making changes to the website files, we navigated to the project folder, right-clicked inside the folder, and selected "Open Git Bash here."

Here you can see again how to open the Git Bash program for your folder

  1. Staged the changes: We used the following command to add all modified files to the staging area: 
    git add .
  2. Committed the changes: To keep track of what was updated, we wrote a descriptive commit message using the following command: 
    git commit -m "Updated the website layout and added a new section"
  3. Pushed the changes: Once committed, we pushed the updates to the GitHub repository using: 
    git push
  4. Verified the updates: Finally, we visited the live site to confirm that all the changes had been successfully applied.

With this process, we could update the website and upload it again, without too much fuss.

Final Words

This assignment was a great introduction to building websites and getting hands-on experience with tools like Git and GitHub. It taught me how to structure a project, manage files efficiently, and publish a website live for anyone to see. While it was challenging at times—especially figuring out Git commands—it felt really rewarding to see the website come together.

Creating this website gave me a solid foundation for a portfolio that I can use to store and display the assignments I’ll be working on next. It’s satisfying to have a central place to organize everything and show my progress throughout the course.
